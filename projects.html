Timeline

Day 1 - 6/8/18 - Getting Started ▾


  • Obtained robot and other equipment from the lab
  • Cleared workspace for optimal robot use

Day 2 - 6/11/18 - Software Installation, Joysticks, and Talons ▾


  • Installed Java, Eclipse, Phoenix, and FRC Driver Station to main computer
  • Tested and debuggd basic programs with roboRIO
  • Removed JDK 9 in favor of the supported JDK 8.
  • Created Drivetrain subsystem and ArcadeDriveVelocity to move
  • Cannot send constant to the TalonSRX: must use joysticks due to update frequency
  • Followed Talon SRX Manual up to part 7

Day 3 - 6/12/18 - SmartDashboard, Configuring Velocity PID ▾


  • Installed SmartDashboard and learned how to print and graph data from code
  • Configured basic settings for velocity PID using the Talon SRX Manual
  • Tested on an elevated platform to start
  • Aligned sensor phase with Talons, configured nominal output and peak output in both directions
  • Created GitHub to store code

Day 4 - 6/13/18 - Mastering Velocity and Position PID ▾


  • Continued Velocity PID, tuning F, P, and D constants to minimize error
  • Later removed robot from blocks and retuned constants
  • Began Position PID, carefully tuning P, I, and D constants using the joystick as a position input
  • Researched optimal carpet to most accurately simulate field conditions
  • Successfully transported the switch from the lab to our workspace

Day 5 - 6/14/18 - Elevator PID and Intake ▾


  • Added current limit to drive train, completed position PID
  • Improved code design with easy-to-read comments and encapsulation
  • Completed pneumatics control
  • Tested compressor with pressure switch
  • Actuated solenoids for compression and intake raising using joystick D-Pad
  • Implemented working Elevator subsystem
  • Created commands to move the elevator given a set position or velocity
  • Created 5-amp current limit to minimize damage
  • Configured sensor phase to align with motors
  • Began elevator position PID
  • Tuned ArbitraryFeedForward constant to counteract effects of gravity
  • Began tuning P constant until damaging chain with poorly selected range
  • Continued Intake subsystem to add cube intaking functionality
  • Successfully implemented left half of intake

Day 6 - 6/15/18 - Wrapping Up Teleop ▾


  • Repaired elevator chain after damaging a link during yesterday's tuning
  • Completed code for both sides of the intake, swapping in a new 775 pro motor for the right half due to a damaged header
  • Borrowed a one-third field carpet to optimize robot performance from Team 115
  • Completed tuning of position PID by setting soft limits and preventing the joystick from zeroing to prevent damage during tuning
  • Further tuning and research is necessary for the arbitrary feed forward constant intended to counteract gravity
  • Incorporated everything into a single controller, animating the robot with a fully functional drive train, elevator, and intake

Day 7 - 6/16/18 - Moving to Command-Based Architecture ▾


  • Created a wrapper class to encapsulate the gamepad
  • Discussed pros and cons of various programming systems, such as command-based and state-based
  • Began to port existing features over to a command-based system with a scheduler
  • Created preset elevator positions using gamepad buttons
